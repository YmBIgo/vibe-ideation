# vibe ideation PoC の使い方

## これは何か？
企業が持っている新しい素材を、世の中にある様々な製品のどこで活用できるかを提案するAI

## なぜ作ったか？
企業が持っている新しい素材が、新しい使い道を見つけるまでの道のりは長く険しい。
人１人が見れ把握できる製造現場の知識には限りがあり、どうしてもランダムにどこが合うか考えがちである。

しかし、今は「AI」という自然言語を操れる存在がある。
「新素材を活かせる場所を探す」という険しい道を「AI」がどのようにして効率化できるかを考え、「vibe coding」ならぬ「vibe ideation」のPoCを作った。

## 使い方（単発）

1. [下準備] nodeのインストール

```bash:nodeのインストール
# for Linux
apt get node
# for Mac
brew install node
```

2. [下準備] OpenAIのAPI Key取得

https://qiita.com/kurata04/items/a10bdc44cc0d1e62dad3

などを参考にしてAPI Keyを取得して、
環境変数 "OPENAI_API_KEY" にそのAPI Keyを設定

3. nodeアプリの環境設定

```bash
# 今のディレクトリは node とします
npm i
```

4. nodeアプリの実行

```bash
npm run start
```

を入力すると、

```bash
「調べたい製品を入力してください。」
```

が出てくるので、答えます。

```bash
EV（電気自動車）
```

製品名を入力すると、次の質問が出ます。

```bash
応用したい素材の名前を簡潔に入力してください。
応用したい素材の特徴を入力してください
```

ここで保有している新素材を入力します。
すると、OpenAI APIを活用した自動探索スクリプトが走り、
以下のJSONファイルが40〜1時間で作られます（1ドル程度）。

・json/素材名/products/製品名.json
　製品の動作の流れの説明
・json/素材名/competitor/製品名/動作.json
　現状の動作を担保する素材の情報
・json/素材名/idea/製品名/動作名/競合名.json
　競合に対する新しいアイデアの案

全てできていなくても、途中経過はJSONで見れるので、以下でgrep（VSCodeなどを使用）できます。
もちろん、grepは全て完了してから行っても大丈夫です。

```grep:「88」部分は点数を書く
"score": 88
```

## 使い方（100回バッチ処理）

> 注意
> こちらは一回APIを回し切ると100ドル程度のAPI費用がかかります
> 利用には気をつけてください

#### 概要
こちらは、上記の単体でのアイデア生成を主要な家電100個で回すスクリプトです。
総当たりで新素材の使い道を探す時に使えます。
使い方としては、一度スクリプトを回し、その結果のJSONから「score」が高いものを精査する形になります。

---

1. [下準備] nodeのインストール

```bash:nodeのインストール
# for Linux
apt get node
# for Mac
brew install node
```

2. [下準備] OpenAIのAPI Key取得

https://qiita.com/kurata04/items/a10bdc44cc0d1e62dad3

などを参考にしてAPI Keyを取得して、
環境変数 "OPENAI_API_KEY" にそのAPI Keyを設定

3. nodeアプリの環境設定

```bash
# 今のディレクトリは node とします
npm i
```

jsonフォルダをこのレポジトリのルートフォルダ配下に作り、
sample/products_kaden.json を、jsonフォルダ配下におきます。

4. nodeアプリの実行

```bash
npm run loop
```

を入力すると、質問が出ます。

```bash
応用したい素材の名前を簡潔に入力してください。
応用したい素材の特徴を入力してください
```

ここで保有している新素材を入力します。
すると、OpenAI APIを活用した自動探索スクリプトが走り、
以下のJSONファイルが20〜40分で作られます（100ドル程度）。

・json/素材名/products/製品名.json
　製品の動作の流れの説明
・json/素材名/competitor/製品名/動作.json
　現状の動作を担保する素材の情報
・json/素材名/idea/製品名/動作名/競合名.json
　競合に対する新しいアイデアの案

全てできていなくても、途中経過はJSONで見れるので、以下でgrep（VSCodeなどを使用）できます。
